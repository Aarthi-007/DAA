What is Big-O Notation?

Definition

Big-O notation describes the upper bound of an algorithm's running time as input size grows toward infinity. It tells you the worst-case scenario — how slow can this get?

Why it matters

  • Predict how code behaves at scale
  • Compare competing solutions objectively
  • Optimize bottlenecks before they hurt
  • Pass technical interviews confidently

Key rules

  • Drop constant multipliers: O(3n) → O(n)
  • Keep the dominant term: O(n² + n) → O(n²)
  • Nested loops multiply: O(n) × O(n) = O(n²)
  • Sequential blocks add: O(n) + O(n) = O(n)
